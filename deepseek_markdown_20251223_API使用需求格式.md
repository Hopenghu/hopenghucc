## 2. 用户场景与功能详述
针对每个AI功能点，按以下结构详细描述：

### 功能点 1: [例如：智能导购问答]
*   **用户是谁**： 浏览商品的潜在顾客
*   **用户痛点/需求**： 想快速了解商品细节（如材质、尺寸、保养方式），但不想翻阅冗长的参数页。
*   **预期AI行为**：
    1.  用户在前端界面提问（如：“这件衬衫适合夏天穿吗？”）。
    2.  网站后端将**商品信息（标题、参数）** 和**用户问题**组合成提示词，调用DeepSeek API。
    3.  AI基于提供的商品信息进行回答，回答需**严谨**，不虚构参数表中没有的信息。
    4.  前端友好地展示回答。
*   **成功指标**： 用户针对商品的咨询转化率提升X%；客服相关工单减少Y%。

### 功能点 2: [例如：个性化推荐理由生成]
*   **用户是谁**： 查看“猜你喜欢”模块的用户
*   **用户痛点/需求**： 看到推荐商品列表，但不知为何被推荐，缺乏点击动力。
*   **预期AI行为**：
    1.  后端根据用户浏览历史，生成推荐商品列表。
    2.  针对每个推荐商品，调用DeepSeek API，生成一句**个性化的推荐理由**（如：“因为你浏览过户外帐篷，这款防潮垫与你之前的兴趣高度匹配”）。
    3.  前端在推荐商品旁展示该理由。
*   **成功指标**： “猜你喜欢”模块的点击率提升X%。

## 4. 数据与约束
*   **输入数据来源**： [例如：商品数据库的JSON字段、用户当前的会话历史]
*   **输出限制**：
    *   长度限制：每个回复不超过200字。
    *   内容限制：禁止生成任何营销承诺或医疗建议。

*   **错误处理**： 当API调用失败或超时时，前端统一显示：“服务暂时繁忙，请稍后再试。”
